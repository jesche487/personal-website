What It Does

A real-time hand sign recognition system that detects domain expansion gestures from the anime Jujutsu Kaisen. The app processes a live camera feed, uses MediaPipe to extract 21 hand landmark points per frame, and runs them through a trained ML classifier to identify specific signs — triggering animated GIF overlays for Gojo's Unlimited Void and Sukuna's Malevolent Shrine when detected.

Technical Decisions

  • Used MediaPipe HandLandmarker to extract 21 normalized landmark coordinates per hand — training on landmark data rather than raw images means the model stays tiny, trains in seconds, and needs no GPU.
  • Started with manual geometric rules to detect Gojo's sign (finger angles, relative positions) to get the full pipeline working first, then swapped in a scikit-learn classifier so adding new signs just means recording more samples and retraining.
  • OpenCV handles the split-screen display: left half shows the live camera feed with the hand skeleton drawn, right half plays the domain expansion GIF when a sign is detected.
  • GIFs are stored in a dictionary keyed by sign name — adding a new domain expansion requires no structural changes, just a new entry and a set of training samples.
  • Training data is collected via a separate calibrate.py script: press space to record samples of a sign, save to a .npy file, then run train.py to produce a new model.

Problems Solved

  • MediaPipe 0.10+ deprecated the mp.solutions.hands API — migrated to the new HandLandmarker tasks API and handled the model download on first run.
  • OpenCV windows weren't appearing on Windows — tracked it down to a thread timing issue and added explicit window focus logic.
  • GIF playback required frame-by-frame management in OpenCV since it has no native GIF support — built a small loader that cycles frames at the correct interval.

What I'd Do Differently / Next

  • Add more domain expansions — the ML pipeline makes this straightforward, just record samples and retrain.
  • Improve confidence thresholding so detections require holding a sign for a brief duration, reducing false positives.
  • Package it as a standalone executable so it runs without a Python environment.
  • Explore adding two-hand gesture support for signs that use both hands simultaneously.
